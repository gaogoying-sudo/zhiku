# 单菜热过程分析 / 温度预测 PRD v0.1

更新时间：2026-06-07

## 背景

起火治理不能只看单点温度。当前更需要把一 道菜的设计动作、真实功率输出、投油/投主料窗口、投料吸热、实测锅温偏差和设备状态放到同一页，形成后续预测锅内真实温度和安全风险治理的工作台。

## 一句话目标

针对单台设备、单次烹饪作业，展示：

```text
理论能量输入 -> 投料热负载 -> 预测锅温变化 -> 实测锅温偏差 -> 安全风险窗口
```

## 页面入口

新增页面：`日志诊断 / 热过程分析`

它位于：

```text
温度诊断 -> 功率诊断 -> 热过程分析
```

三者分工：

- 温度诊断：对齐菜谱步骤、真实动作和锅体温度。
- 功率诊断：核对指令功率、实际输出功率和能量积分。
- 热过程分析：把功率能量、投料吸热、理论锅温和实测偏差放到一页，服务温度预测和起火风险治理。

## 第一版能力

1. 复用现有日志包多选、下载、解压和解析链路。
2. 支持选择单道菜 / 单次作业。
3. 支持“包含当前作业内热锅/润锅”开关。
4. 展示模型参数：
   - 室温 / 食材初温
   - 热效率
   - 锅体等效热容
   - 散热系数
   - 初始锅温
5. 展示能量总览：
   - 实际输出能量
   - 指令能量
   - 预测最高锅温
   - 实测最高锅温
6. 展示曲线：
   - 理论预测锅温
   - 实测锅温 / 输出温度
   - 实际输出功率
   - 指令功率
   - 投油、主料、热锅节点
7. 展示动作链阶段与能量窗口。
8. 从步骤和安卓动作文本中抽取带重量的投料，估算食材热负载。
9. 输出预测/实测偏差解释。
10. 输出安全风险标签：
    - 投油到主料间隔过长
    - 主料前累计能量过高
    - 实测高温
    - 预测高温
    - 缺功率采样
    - 缺实测温度
    - 预测/实测偏差过大

## 第一版计算口径

能量积分：

```text
E_kWh = Σ(P_kW × Δt_s / 3600)
E_kJ = E_kWh × 3600
```

功率优先级：

```text
actual_power_kw 优先
缺失时退回 command_power_kw
```

食材吸热估算：

```text
Q_food = m_kg × c_kJ_kg_C × (T_target - T_room)
```

预测锅温离散更新：

```text
T_next = T_current
  + (P_actual × η × Δt - Q_loss - Q_food_in) / C_system
```

其中：

```text
Q_loss = k_loss × max(0, T_current - T_room) × Δt
C_system = C_pan + Σ(m_food_in_pan × c_food)
```

第一版默认值：

- 室温 / 食材初温：25°C
- 热效率：0.85
- 锅体等效热容：3.2 kJ/°C
- 散热系数：0.012 kJ/s/°C

## 当前限制

- 未接入菜谱和外部润锅/热锅关联字段表；当前仅识别单次作业窗口内的热锅/润锅动作。
- 未接入真实锅型、锅龄、锅黑等级、红外状态、换锅/换机芯记录。
- 食材比热容为默认分类值，不是精确物性表。
- 投料识别基于步骤文本和安卓动作文本，缺少配料表时可能漏识别。
- 预测曲线是工程近似，不可作为最终安全阈值。

## 后续演进

1. 接入菜谱与润锅关联表。
2. 接入标准配料表、食材类别、比热容、含水率和汽化潜热。
3. 接入设备锅状态：
   - 锅型
   - 锅龄
   - 锅黑等级
   - 换锅记录
   - 红外状态
   - 机芯/线盘版本
4. 建立同菜谱、同设备、多日期偏差样本。
5. 增加参数拟合：
   - 热效率
   - 等效热容
   - 散热系数
   - 红外偏差
6. 把高风险规则沉淀为后台扫描：
   - 投油前累计能量
   - 投油后主料延迟
   - 高功率空烧
   - 预测/实测偏差
   - 测温可信度不足

## 验收标准

1. 查询任意设备后，能进入 `热过程分析` 页面。
2. 选择日志包并解析后，可以选择某一道菜。
3. 页面能展示实际/指令能量、预测最高锅温、实测最高锅温。
4. 页面能展示预测温度、实测温度、实际功率、指令功率曲线。
5. 投油、主料、热锅节点能在曲线上标记。
6. 能看到动作链阶段和每阶段能量。
7. 能看到从步骤文本识别出来的投料热负载。
8. 能看到偏差解释和安全风险标签。
9. 关闭“包含当前作业内热锅/润锅”后，总能量和预测曲线会按非热锅阶段重新计算。
