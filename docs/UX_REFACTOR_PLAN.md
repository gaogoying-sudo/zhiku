# 智库设备风险排查体验重构

更新时间：2026-06-14 CST

## 1. 目标

智库从“按技术模块堆叠功能”调整为围绕单台设备的连续排查工作流：

`SN → 设备风险总览 → 作业列表 → 作业热安全分析 → 日志证据 → 菜谱/风险/导出`

核心原则：

- SN 是全局一级上下文，日期范围是二级上下文。
- 单次作业是温度、功率、动作、投料和风险证据的共同对象。
- 日志包是证据来源和失败治理对象，不是日常分析的第一入口。
- 数据优先读取云端结构化库和成功缓存，缺失时才下载 ZIP 解析。
- 风险均为候选风险和待复核证据，不输出确定事故结论。

## 2. 信息架构

### 工作台

- `0 工作台总览`：当前设备状态、最近设备、后台任务和快捷入口。

### 单设备排查

- `1 设备风险总览`：档案、结构化覆盖率、候选风险和下一步建议。
- `2 作业列表`：按日期读取结构化作业，筛选高温、高功率、告警和 DB 未匹配作业。
- `3 作业热安全分析`：同屏查看温度、功率、能量、动作、步骤和原始证据。
- `4 日志证据中心`：日志包状态、失败原因、内部作业和批量补齐。
- `5 菜谱与资源`：菜谱资源覆盖，并反查关联作业和风险。

### 全局检索

- `R 菜谱检索`
- `S 安全总览`
- `△ 告警明细`
- `! 重点设备`

### 治理工具

- `V 版本治理`
- `K 热物性库`
- `C 温度校正`
- `A 系统审计`

### 调试入口

- `D 原日志结论`
- `T 原温度诊断`
- `P 原功率诊断`
- `H 原热过程分析`

旧能力继续保留，主流程不再要求用户理解底层日志解析模块。

## 3. 全局上下文

主页面共享：

- 当前 SN、客户、地区、版本。
- 当前开始/结束日期。
- 数据来源：结构化库、服务端缓存、浏览器缓存、实时解析或源数据库。
- 日志入库进度、候选风险等级、最近更新时间。

统一动作：

- 刷新当前数据。
- 补齐最近日志。
- 进入热安全分析。
- 导出当前证据。
- 清除当前 SN 缓存。

## 4. 主要用户流程

1. 查询或从历史设备选择 SN。
2. 默认进入设备风险总览，查看数据完整度和候选风险。
3. 点击指标或下一步建议进入已带筛选条件的作业列表。
4. 从单次作业进入热安全分析，结构化作业优先。
5. 证据不足时反查关联日志包，或在日志证据中心批量补齐。
6. 从作业、菜谱或安全告警之间双向跳转。
7. 导出当前设备或单次作业证据。

## 6. 状态与失败规范

日志包状态应区分：

- 可下载、已下载、排队中、解析中、已解析、已入库、部分成功。
- DB 未匹配但日志内有作业。
- DB 未匹配且日志内无有效作业。
- 关键日志缺失、下载失败、解压失败、解析失败、入库失败、远端不可用。

`已入库` 只表示结构化结果已写入本地分析库；DB 作业匹配是独立状态。DB 未匹配不等于未生产。

失败对象统一返回：

```json
{
  "stage": "",
  "failure_reason": "",
  "message": "",
  "suggested_action": "",
  "can_retry": true,
  "retry_action": "",
  "data_available": true
}
```

失败阶段包括源库查询、日志索引、下载、解压、文件识别、Android/温度/功率/作业解析、DB 匹配、MySQL 入库、结构化读取和前端渲染。

## 7. 按钮命名

按钮说明动作和对象，例如：

- `进入热安全分析`
- `查看日志证据`
- `查看包体诊断`
- `批量解析入库`
- `补齐最近日志`
- `读取结构化数据`
- `下载原包`
- `导出当前证据`
- `清除当前 SN 缓存`

避免单独使用“分析、处理、查看、刷新”等边界不清的文案。

## 8. 本轮实现

- 重排全站导航和主流程，旧诊断页降级为调试入口。
- 增加全局 SN、日期、来源、入库进度、候选风险和更新时间上下文条。
- 新增本地结构化作业接口 `GET /api/device-jobs/{sn}`。
- 设备风险总览增加结构化覆盖率、温度/功率/告警摘要和可执行建议。
- 作业列表支持结构化库优先、异常筛选及作业到热安全/日志证据的跳转。
- 日志证据中心支持多选、大小限制、批量诊断、失败跳过和批量解析入库。
- 热安全分析支持优先选择结构化作业，并保留日志包证据回退。
- 菜谱、安全总览、设备、作业和日志证据之间建立关联跳转。
- 修正“已入库”和“DB 已匹配”的状态混用。
